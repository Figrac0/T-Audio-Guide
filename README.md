## Что такое T-Guide

T-Guide — это веб-приложение для городских аудиоэкскурсий. Пользователь видит карту с точками интереса вокруг себя, может прослушать аудиогид к каждому месту, прочитать транскрипт, добавить точки в личный маршрут и пройти его с пошаговой навигацией. Готовые кураторские маршруты доступны сразу — от архитектурных прогулок до гастрономических туров.

---

## Возможности

### 🗺️ Интерактивная карта открытий

Главный экран — карта на весь экран с маркерами ближайших мест. Радиус поиска настраивается. Точки фильтруются по категориям: **Музеи · Развлечения · История · Еда · Природа**. Снизу — шторка с drag-to-snap (три позиции: скрыта / peek / полная), которая показывает карточки мест и каталог маршрутов.

При клике на маркер карта плавно центрируется на точке, шторка раскрывается с мини-карточкой. Геолокация определяется автоматически; если нет доступа — можно задать позицию вручную прямо на карте.

---

### 🎧 Аудиогид

Каждая точка интереса может содержать аудиозапись с профессиональным рассказом о месте. В панели точки:

- Кнопка **«Прослушать»** — запускает аудио через нативный HTML5 Audio API
- Отображается длительность записи (загружается через `preload="metadata"`)
- Язык аудиогида показан отдельно
- Кнопка **«Прочитать»** — раскрывает полный текст транскрипта с анимированной высотой
- Транскрипт разбит на параграфы, удобен для людей с нарушениями слуха

---

### 🧭 Конструктор маршрутов

На странице `/excursions` пользователь собирает маршрут вручную:

1. Карта показывает ближайшие точки с маркерами и цветовой индикацией категории
2. Клик по маркеру → панель подробностей с фото, описанием, метриками и аудиогидом
3. Кнопка **«Добавить в маршрут»** → точка попадает в черновик
4. Черновик отображается на карте как нумерованная цепочка
5. Кнопка **«Сохранить маршрут»** → маршрут появляется в профиле

Лимит черновика — 10 точек. Порядок точек меняется перетаскиванием. Маршрут автоматически строится по дорогам через OSRM API.

---

### 🚶 Навигация по маршруту

После старта экскурсии приложение переходит в режим навигации:

- **Карта** — на весь экран с маркером текущей позиции пользователя и выделенной остановкой
- **Расстояние до точки** — обновляется в реальном времени по GPS
- **Время пешего хода** — рассчитывается автоматически (~5 км/ч)
- **Шторка** с drag-to-snap открывает карточку текущей остановки:
  - Фото, название, категория
  - Часы работы, рейтинг, время на осмотр
  - Полное описание о месте
  - Аудиогид с транскриптом
- Кнопки **«Предыдущая» / «Следующая»** — переход между остановками
- При смене остановки карта плавно центрируется, аудио и транскрипт сбрасываются
- Кнопка **«Завершить»** — маршрут фиксируется в истории профиля

---

### 📋 Готовые маршруты

Кураторские маршруты от редакции T-Guide — доступны на главной и в каталоге. Каждый маршрут содержит:

- Обложку с темой и сложностью (Легко / Средне / Сложно — по длине маршрута)
- Краткое и полное описание
- Список всех остановок с превью
- Общую длину, время и количество точек
- Рейтинг и количество отзывов

Фильтрация каталога: по теме (архитектура, еда, природа, история, развлечения) и продолжительности.

---

### 👤 Личный кабинет

Страница `/profile` — персональный хаб пользователя:

| Раздел | Содержимое |
|--------|-----------|
| **Профиль** | Имя, email, язык интерфейса (ru / en), смена пароля |
| **Сохранённые маршруты** | Маршруты, добавленные в избранное одной кнопкой |
| **Мои маршруты** | Личные маршруты, созданные в конструкторе |
| **История** | Пройденные экскурсии с датой и процентом завершения |

Сохранённые и личные маршруты открываются прямо из профиля — одним тапом переход в навигацию.

---

### 🔐 Аутентификация

- Регистрация по email + пароль
- Вход с JWT-токенами (access + refresh)
- Автоматическое обновление access-токена при 401
- Смена пароля из профиля
- Восстановление пароля по email
- Защищённые маршруты с редиректом и возвратом на исходную страницу после входа

---

### ⚙️ Административная панель

Доступна по адресу `/admin` для пользователей с ролью `admin`. Полнофункциональный CRUD для всего контента приложения:

**Точки интереса**
- Создание / редактирование / удаление точек
- Поиск по названию с пагинацией
- Поля: название, описание, краткое описание, категория, координаты, адрес, время на осмотр, часы работы, статус активности
- Интерактивная карта для выбора координат с геокодером (поиск по адресу через Nominatim)
- Загрузка медиа: **фото, аудио, видео** — прямо при создании точки
- Управление порядком медиа (sortOrder), транскрипт для аудио

**Категории**
- Создание, переименование, удаление категорий с проверкой зависимостей

**Экскурсии**
- Создание кураторских маршрутов с подбором точек
- Управление порядком точек в маршруте
- Настройка видимости (публичный / приватный)

**Пользователи**
- Список всех пользователей с поиском и пагинацией
- Смена роли (user / admin)
- Активация / блокировка аккаунтов

## Архитектура

Проект построен по методологии **Feature-Sliced Design (FSD)**:

```
src/
├── app/              ← Точка входа, роутер, провайдеры (Auth, UserRoutes)
├── pages/            ← Страницы: Home, Excursions, Excursion, Profile, Admin, SignIn
├── features/         ← Функциональные модули: карта (Leaflet), геолокация, пользовательские маршруты
├── entities/         ← Доменные модели: Excursion, NearbyPoint, RouteStop, хуки данных
├── widgets/          ← Составные блоки: ExcursionCatalog, RouteOverview
└── shared/           ← API-слой, утилиты, конфиги, токены, UI-компоненты
```

### Технологический стек

| Технология | Версия | Роль |
|-----------|--------|------|
| React | 19.2 | UI-фреймворк |
| TypeScript | 5.9 | Типобезопасность |
| Vite | 8.0 | Сборка и HMR |
| React Router | 7.13 | Клиентский роутинг |
| Leaflet | 1.9 | Интерактивные карты (OSM) |
| Capacitor | 8 | Нативная Android-оболочка |
| CSS (global) | — | Без CSS-in-JS, дизайн-токены через переменные |

### Ключевые паттерны

- **Session-level кеш** точек — `/points/{id}` запрашивается один раз за сессию, обогащение через `usePointDetailsMap`
- **Drag-to-snap шторка** — собственная реализация без библиотек, работает на touch и mouse
- **Lazy loading** — все страницы загружаются по требованию через `React.lazy`
- **Refresh-token flow** — прозрачное обновление JWT при 401 с повтором исходного запроса
- **OSRM** — построение пешего маршрута по дорогам между точками

## Развёртывание

### Docker (рекомендуется)

Образ публикуется автоматически в GitHub Container Registry при каждом пуше в `main`:

```bash
docker pull ghcr.io/figrac0/t-guide:latest

docker run -d \
  --name t-guide \
  -p 3000:80 \
  --restart unless-stopped \
  ghcr.io/figrac0/t-guide:latest
```

Приложение доступно на `http://localhost:3000`.

Для обновления до новой версии:

```bash
docker pull ghcr.io/figrac0/t-guide:latest && docker restart t-guide
```

### Сборка с кастомным API

```bash
docker build \
  --build-arg VITE_API_URL=https://your-api.example.com/api \
  --build-arg VITE_MAP_PROVIDER=osm \
  -t t-guide .

docker run -d --name t-guide -p 3000:80 t-guide
```

### Локальная разработка

```bash
git clone https://github.com/Figrac0/T-Audio-Guide.git
cd T-Audio-Guide
npm install
cp .env.example .env   # задать VITE_API_URL
npm run dev
```

### Android (нативное приложение)

T-Guide распространяется и как нативное Android-приложение через [Capacitor 8](https://capacitorjs.com/). Веб-сборка упаковывается в APK без изменения кода — все функции работают идентично веб-версии.

**Что работает нативно:**
- GPS-геолокация через системный FusedLocationProvider (быстрее и точнее браузерного API)
- Edge-to-edge layout — хедер покрывает зону статус-бара, жесты навигации не перекрывают контент
- Нативный спиннер разрешений (`requestPermissions`) при первом запросе геолокации

**Сборка и запуск:**

```bash
# 1. Собрать веб-часть
npm run build

# 2. Синхронизировать в Android-проект
npx cap sync android

# 3. Открыть в Android Studio и запустить на устройстве или эмуляторе
npx cap open android
```

После `cap sync` все изменения из веб-версии (UI, логика, стили) автоматически попадают в Android-приложение — пересобирать APK вручную не нужно, достаточно нажать Run в Android Studio.

## CI/CD

GitHub Actions автоматически собирает и публикует Docker-образ при пуше в `main`:

```
push → main
  └── Checkout
  └── Docker Buildx
  └── Login GHCR
  └── Build & Push → ghcr.io/figrac0/t-guide:latest
                   → ghcr.io/figrac0/t-guide:{sha}
```

Кеширование слоёв через GitHub Actions Cache (`type=gha`) ускоряет повторные сборки.
